Verifiable Answer UI — 성과관리 AI Agent 재설계

성과관리/평가 E2E 에이전트의 모든 답을 검증 가능한 객체(answer receipt)로 렌더링한다. 하나의 에이전트가 상황에 따라 도킹 대화창 · 선제 팝업 · 전체화면 세 형태로만 화면을 바꾸며, 어느 형태에서 열려도 4개 시그널 레이어를 동일하게 노출한다. 주황색 번호가 4개 레이어의 위치다.

1기준 시점 칩bitemporal as-of 셀렉터. 과거 시점(평가 시즌·마감 스냅샷) 기준 재조회 진입점.
2계산·근거 트레이스규칙 버전 배지, 규정 칩, 원천 레코드 식별자(goal.2026H1.kr3)를 펼침.
3감사 가능 배지감사 로그 기록 여부, 권한 내 탐색 범위 명시.
4What-if 재계산계산 엔진 실증 — 아래 버튼은 실제로 동작합니다.

하나의 에이전트, 상황에 맞춰 화면만 바꾸는 3가지 노출 형태

세 화면은 서로 다른 앱이 아니다. 기본값은 화면 흐름을 끊지 않는 ① 도킹 대화창, 필요할 때 ②·③으로 승격한다. 어떤 모드에서 열려도 같은 확인 내역 · 같은 근거 인용 · 같은 승인/수정/보류 게이트를 공유한다.

기본 · 상주

① 도킹 대화창

화면 우측 340px에 접혀 상주. 평상시 질문·확인·간단 제안을 맥락 안에서 즉시. 화면 전환 없이 대화.

보조 · 선제

② 선제 팝업

기한·편향·이상치 감지 시 먼저 말 건다. "캘리 3일 전, 미제출 4명" 같은 작은 카드로 행동 유도.

심층 · 전환

③ 전체화면 딥워크

전체 화면을 내줘 병렬 심의·다중 소스·대형 객체를 한 자리에. 여러 사람 데이터를 펼쳐 집중 작업.

3모드 공통 계약 확인 내역 근거 인용 승인 / 수정 / 보류 게이트
형태 ①

도킹 대화창 — 보던 화면 위에 답변 영수증을 얹는다
