# elizax_talenx 용어집 (UI 카피 표준)

> 검수 기준: **"HR 담당자가 옆자리 동료에게 설명 없이 쓸 수 있는 말인가."**
>
> 화면·토스트·배지·에이전트 말풍선 등 **사용자에게 보이는 모든 문자열**은 이 표의 "확정 용어"만 사용한다.
> 시스템 설계 용어(왼쪽 열)는 설계 문서·데모 내레이션·코드 식별자에만 남긴다.
> 함수명·CSS 클래스·데이터 키 등 **식별자는 개명하지 않는다** (예: `EZLedger`, `.agh-b-suggest`, `human_approve` 모드 키).

## 확정 용어

| 시스템/설계 용어 (내부) | 확정 UI 용어 | 비고 |
|---|---|---|
| 전체화면 딥워크 | **워크스페이스** | Hub 헤더·툴팁 |
| 맥락 원장 / 원장 축적 (Context Ledger) | **성과 히스토리** / 히스토리에 기록됨 | 패널 제목·배지·토스트 |
| 초안 객체화 (완료) | **초안 생성됨** | 진행 상태 라인·토스트 |
| 근거 정초 | **근거 확인 완료** | 진행 상태 라인 |
| 자산화 / 산출물 자산화 | **기록 보관됨 / 기록으로 보관** | 게이트 토스트·산출물 화면 |
| 게이팅 (S7) | **승인 대기** | UI에는 "결정 게이트 · 사람이 확정" 유지 |
| `auto` / `suggest` / `human_approve` (칩 노출) | **자동 처리 / 제안만 / 승인 필요** | 데이터 키·CSS 클래스는 영문 유지 |
| (목표) 정합성 (점검) | **목표 정렬 점검 / 정렬·중복 점검** | 현업 표준어 Alignment |
| as-of / 스냅샷 | **기준 시점** | 전 화면 통일 |
| 탈렌트 세션 | **인재 리뷰** | 평가관리 서브탭·카드·확인창 |
| S1~S8 프로토콜 라벨 | (UI 노출 금지) | README·데모 내레이션 전용 |

## 문체 원칙 (에이전트 응답 포함)

1. 결정·상태는 사람이 하는 행위로 쓴다 — "승인 대기", "조직장이 확정합니다" (시스템 상태어 금지).
2. 근거는 실무자가 찾는 이름으로 부른다 — "체크인 기록", "평가규정 v3.1", "직무 과업".
3. 영문 코드·개발 관용어(딥워크, ledger, 객체화)는 화면에 내보내지 않는다.
4. 새 기능의 카피를 쓸 때 이 표에 없는 내부 용어가 생기면, 먼저 이 표에 확정 용어를 추가한 뒤 사용한다.
